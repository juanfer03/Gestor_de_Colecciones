# Fase 1 — Documentación y Maquetación (HTML + CSS)

## Objetivo

Crear toda la documentación inicial y una maqueta estática completa (HTML + CSS) que represente la galería de libros y tarjetas, responsive y accesible.

## Subfases y tareas

- 1.1 Documentación: completar `docs/alcance_del_proyecto.md`, `docs/arquitectura_del_proyecto.md`, `docs/tech_stack.md` (esqueleto) y plantillas.
- 1.2 Sistema de diseño estático: definir paleta, tipografías y componentes visuales básicos en `docs/sistema_de_diseño.md` (esqueleto + ejemplos CSS).
- 1.3 Maquetación: crear páginas HTML estáticas:
  - `index.html` (galería)
  - `book.html` (detalle tarjeta)
  - Formularios básicos (agregar/editar libro)
- 1.4 Accesibilidad y responsive: pruebas con Lighthouse y correcciones.
- 1.5 Entrega: añadir archivos estáticos al repositorio y actualizar README con enlaces a los recursos.

## Criterios de aceptación Fase 1

- Documentos requeridos creados y revisados.
- HTML/CSS responsive con galería y tarjetas funcionales.
- Cumple WCAG 2.1 AA en los puntos críticos (contraste, navegación teclado, alt en imágenes).

## Artefactos entregables

- `docs/` actualizados.
- Carpeta `static_prototype/` con `index.html`, `styles.css`, assets.
